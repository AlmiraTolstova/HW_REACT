Часть 1
Создайте store и редьюсер для управления каталогом книг. Книга описывается объектом:
{
id: number | string,
title: string,
author: string,
year: number,
isAvailable: boolean // доступна ли книга для выдачи
}
Требуемые action types и их логика:
1.BOOK_ADD - добавление новой книги
Принимает объект книги БЕЗ id и isAvailable
Генерирует уникальный id (можно через Date.now() + случайное число)
Устанавливает isAvailable: true по умолчанию
2.BOOK_REMOVE - удаление книги по id
Если книга сейчас выдана (isAvailable: false), удалять её нельзя! (проверка)
3.BOOK_UPDATE_INFO - обновление информации о книге
Можно менять только title, author, year
Нельзя менять id и isAvailable через этот экшен
4.BOOK_TOGGLE_AVAILABILITY - изменение статуса доступности
Просто переключает isAvailable на противоположное значение
Структура состояния:
{
books: [], // массив книг
lastUpdated: null // строка с датой последнего изменения (new Date().toISOString())
}
Проверка в консоли:
Добавьте книгу "Гарри Поттер", Дж. Ролинг, 1967
Добавьте книгу "1984", Оруэлл, 1949
Попробуйте удалить первую книгу (должно удалиться, т.к. она доступна)
Обновите год у "1984" на 1948
Переключите доступность у оставшейся книги

Task 2
асть 2
Расширьте приложение. Теперь нужно добавить читателей и привязать к ним книги.

Новая структура состояния:
{
books: [], // массив книг (из Этапа 1)
readers: [ // новый массив читателей
{
id: number,
name: string,
email: string,
borrowedBooks: [] // массив id книг, которые взял читатель
}
]
}

Новые action types:
1.READER_ADD - регистрация нового читателя
Принимает name и email
Генерирует id
Создает пустой массив borrowedBooks
2.READER_REMOVE - удаление читателя
Можно удалить только если у читателя нет книг (borrowedBooks.length === 0)
3.BOOK_LEND_TO_READER - выдача книги читателю
Принимает bookId и readerId
Условия успешной выдачи:
Книга должна существовать и быть доступна (isAvailable: true)
Читатель должен существовать
При успехе:
У книги isAvailable становится false
В массив borrowedBooks читателя добавляется bookId
4.BOOK_RETURN_FROM_READER - возврат книги
Принимает bookId и readerId
Условия:
Книга должна быть у этого читателя
При успехе:
У книги isAvailable становится true
bookId удаляется из borrowedBooks читателя

Проверка в консоли:
Добавьте читателя "Иван Петров", "ivan@mail.com"
Добавьте читателя "Анна Смирнова", "anna@mail.com"
Выдайте книгу "1984" Ивану
Попробуйте выдать ту же книгу Анне (должна быть ошибка или ничего не произойти)
Верните книгу от Ивана
Выдайте книгу Анне (теперь должно получиться)

Task3
Часть 3
Добавить в состояние статистические данные, которые будут автоматически пересчитываться в редьюсерах при любых изменениях книг или читателей.

Финальная структура состояния:
{
books: [],
readers: [],
statistics: { // НОВОЕ ПОЛЕ
totalBooks: 0, // всего книг
availableBooks: 0, // доступно сейчас
borrowedBooks: 0, // выдано всего
booksByDecade: { // книги по десятилетиям
'1950': 0,
'1960': 0,
'1970': 0,
// и т.д.
},
activeReadersCount: 0, // читатели с книгами на руках
mostPopularAuthor: { // самый популярный автор
name: '',
booksCount: 0
}
}
}

Модифицируйте ВСЕ существующие редьюсеры так, чтобы после каждого изменения книг или читателей статистика пересчитывалась заново
1.Статистика должна быть всегда актуальной
2.mostPopularAuthor нужно вычислять на основе количества книг автора в библиотеке (не учитываем, выданы они или нет)
4.booksByDecade должно автоматически создавать новые десятилетия при добавлении книг

\*Добавьте в статистику поле consistencyCheck: boolean, которое проверяет:
1.Сумма availableBooks и borrowedBooks равна totalBooks
2.Количество книг на руках у всех читателей равно borrowedBooks из статистики
Если проверка не проходит - выводите предупреждение в консоль (имитация логирования ошибки).
