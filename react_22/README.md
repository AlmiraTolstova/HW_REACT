1. Подготовка рабочего окружения

Установите Node.js: Убедитесь, что Node.js установлен на вашем компьютере. Это позволит вам использовать npm (Node Package Manager) для управления зависимостями проекта.

Создайте новый проект React: Используйте команду npx create-react-app my-app для создания нового проекта. Поменяйте my-app на название вашего проекта.

Перейдите в каталог проекта: После создания проекта перейдите в каталог вашего проекта, используя команду cd my-app.

Установите нужные пакеты: Установите react-hook-form для работы с формами, используя команду npm install react-hook-form.

2. Создание структуры файлов

Создайте структуру папок для компонентов (components, context, styles): Создайте следующие файлы в папке components: RegistrationForm.jsx, UserProfile.jsx, MainComponent.jsx, App.js. Также создайте файл UserContext.js в папке context.

Обновите App.js: Используйте этот файл для интеграции контекста и рендеринга основного компонента.

3. Определение и настройка контекста

В файле UserContext оздайте контекст с помощью React.createContext(), который позволит делиться состоянием между компонентами.

Напишите провайдер контекста, который будет управлять данными пользователя (примеры полей: логин, email) и статусом регистрации (зарегистрирован пользователь или нет).

4. Имплементация функций провайдера

loginUser(data): Эта функция принимает данные пользователя и обновляет состояния для хранения данных пользователя и статуса его регистрации.

logoutUser(): Функция обнуляет информацию о пользователе и изменяет статус на "не зарегистрирован".

5. Настройка useForm и валидация

В файле RegistrationForm используйте useForm из react-hook-form для управления состоянием формы и валидации.

Валидация полей:

Логин: поле должно быть заполнено.

Email: поле должно быть заполнено и соответствовать формату электронной почты.

Пароль: поле должно быть заполнено и содержать не менее 6 символов.

Функция onSubmit: После успешной валидации, функция onSubmit должна вызывать метод loginUser из контекста и передавать входные данные пользователя.

6. - Настройка UserProfile

Отображение данных пользователя: Используйте контекст для доступа к данным пользователя и их отображения (логин, email).

Функция logout: Создайте кнопку, которая при клике вызывает метод logoutUser из контекста, тем самым деавторизуя пользователя.

7. Основной компонент (MainComponent.js)

Логика отображения: Используя состояние isRegistered из контекста, определите, какой компонент отображать — RegistrationForm или UserProfile.

8. Примеры проверки валидации

Для валидации полей в RegistrationForm.js, вы должны поставить:

Логин: поля должно быть непустым. Это можно указать как { required: true } в методе register.

Email: кроме проверки на непустоту, проверьте соответствие регулярному выражению электронной почты как { required: true, pattern: /^\S+@\S+\.\S+$/ }.

Пароль: должен содержать минимум 6 символов, что можно указать как { required: true, minLength: 6 }.

9. Интеграция и тестирование

Интегрируйте UserProvider в App.js: Обеспечьте доступ к контексту во всем приложении.

Проведите тестирование: Запустите приложение с помощью npm start и проверьте работоспособность регистрации и функций профиля. Убедитесь, что состояние контекста корректно обновляется и доступно в нужных компонентах.
