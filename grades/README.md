initialState
const initialState = {
students: {
"alice@mail.com": {
name: "Алиса",
grades: {
math: [5, 4, 5, 4],
physics: [3, 4, 5],
chemistry: [5, 5]
}
},
"bob@mail.com": {
name: "Боб",
grades: {
math: [2, 3, 4],
physics: [5, 5, 4, 5],
chemistry: [3, 3]
}
},
"carol@mail.com": {
name: "Кэрол",
grades: {
math: [4, 4],
physics: [3, 3, 3]
// chemistry отсутствует (студент ещё не получал оценок по химии)
}
}
},
subjects: ["math", "physics", "chemistry"]
};

students — объект, где ключ = email студента (уникальный идентификатор)
students[email].grades — объект, где ключ = название предмета, значение = массив оценок (числа от 1 до 5)
У студента может не быть предмета в grades (означает, что нет ни одной оценки)
subjects — общий список всех предметов в школе

addGrade
Что делает: добавляет одну оценку студенту по конкретному предмету.
Параметры: payload — объект с полями:
{ email: "alice@mail.com", subject: "math", grade: 5 }

Валидация:
Если студента с таким email нет → ничего не делать (или вывести ошибку в консоль)
Если предмета нет в subjects → ничего не делать
Если grade не число от 1 до 5 включительно → ничего не делать

Логика:
Если у студента ещё нет массива для этого предмета → создать массив [grade]
Если есть → добавить оценку в конец массива

removeLastGrade
Что делает: удаляет последнюю оценку студента по предмету.
Параметры: payload — объект с полями:
{ email: "alice@mail.com", subject: "math" }

Валидация:
Если студента нет → ничего не делать
Если у студента нет оценок по этому предмету → ничего не делать
Если массив оценок пуст (не должно случиться, но на всякий случай) → ничего не делать

Логика:
Удалить последний элемент массива (.pop())
Важно: если после удаления массив стал пустым → удалить ключ предмета из grades (чтобы не хранить пустые массивы)

renameSubject
Что делает: переименовывает предмет во всей системе (у всех студентов и в общем списке).
Параметры: payload — объект с полями:
{ oldName: "physics", newName: "physics" } // исправили опечатку

Валидация:
Если oldName нет в subjects → ничего не делать
Если newName уже есть в subjects → ничего не делать (нельзя создать дубликат)
Если oldName === newName → ничего не делать (бессмысленно)

Логика:
В массиве subjects заменить oldName на newName
Пройти по всем студентам:
Если у студента есть grades[oldName] → перенести массив оценок в grades[newName]
Удалить grades[oldName] (через delete)
Студенты, у которых не было оценок по oldName, ничего не получают (у них не появляется пустой массив по newName)

addStudent
Что делает: добавляет нового студента в журнал.
Параметры: payload — объект с полями:
{ email: "dave@mail.com", name: "Дейв" }

Валидация:
Если студент с таким email уже существует → ничего не делать (не перезаписывать)
Если email или name пустые → ничего не делать

Логика:
Создать запись для студента:
students[email] = {
name: name,
grades: {} // пустой объект, без предметов
}

removeStudent
Что делает: полностью удаляет студента из журнала.
Параметры: payload — строка email (или объект { email } — на выбор студента).

Валидация:
Если студента с таким email нет → ничего не делать

Логика:
Удалить ключ students[email] через delete

addSubject
Что делает: добавляет новый предмет в общий список.
Параметры: payload — строка (название предмета).

Валидация:
Если предмет уже есть в subjects → ничего не делать
Если название пустое → ничего не делать

Логика:
Добавить предмет в конец массива subjects
Важно: у существующих студентов НЕ появляется автоматически массив для нового предмета. Оценки появятся только когда их добавят через addGrade

\*removeSubject
то делает: удаляет предмет из системы.
Параметры: payload — строка (название предмета).

Валидация:
Если предмета нет в subjects → ничего не делать
Если предмет единственный в списке → ничего не делать (нельзя удалить последний предмет)

Логика:
Удалить предмет из массива subjects
Пройти по всем студентам:
Если у студента есть grades[subject] → удалить этот ключ (через delete)

updateStudentName
Что делает: изменяет имя студента (email остаётся ключом, не меняется).
Параметры: payload — объект с полями:
{ email: "alice@mail.com", newName: "Алиса Иванова" }

Валидация:
Если студента нет → ничего не делать
Если newName пустое → ничего не делать

Логика:
Изменить students[email].name
